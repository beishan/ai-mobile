## 1. 硬件规格与选型

### 1.1 主控

| 项目 | 规格 |
|------|------|
| 型号 | ESP32 N16R8（ESP32-D0WD-V3 芯片） |
| Flash | 16MB（四线 SPI） |
| PSRAM | 8MB（八线 OPI PSRAM） |
| 主频 | 240MHz 双核 Xtensa LX6 |
| WiFi | 802.11 b/g/n 2.4GHz |
| 蓝牙 | BLE 4.2 |
| GPIO | 34 个可用 IO，其中 6 个仅输入 |

### 1.2 显示屏

| 项目 | 规格 |
|------|------|
| 尺寸 | 4.2 inch |
| 分辨率 | 400 × 300 px |
| 颜色 | 三色：黑 / 白 / 红|
| 接口 | SPI，时钟 ≤ 4MHz（三色屏刷新速度慢，需降速） |
| 刷新时间 | 全刷约 15–20s（三色屏特性，无局部刷新） |
| 分辨率帧缓冲 | 黑色层 15000 bytes（400×300/8）+ 红色层 15000 bytes，共 30KB |

> **三色屏重要限制**：三色 E-Ink 屏幕**不支持局部刷新**，每次更新必须全刷，刷新时间约 15–20 秒。因此 UI 设计必须减少不必要的刷新次数。游戏等需要频繁更新的功能，每次操作后才刷新一次（非实时）。

### 1.3 MicroSD 卡模块

| 项目 | 规格 |
|------|------|
| 接口 | SPI |
| 文件系统 | FAT32（FATFS）|
| 容量 | ≥ 8GB 推荐 |
| 最大文件 | 单文件 ≤ 4GB（FAT32 限制） |

### 1.4 电源

| 项目 | 规格 |
|------|------|
| 电池 | 3.7V 锂电池，JST-PH 2.0 |
| 充电 | TP4056（USB-C 或 Micro-USB 输入） |
| 电量检测 | 分压电路 → GPIO34（ADC1_CH6），12bit |
| 低电保护 | 电压 < 3.3V 时强制休眠 |

### 1.5 按键

共 4 个物理按键，均为上拉输入，低电平触发：

| 编号 | 推荐 GPIO | 名称 | 深睡唤醒 |
|------|-----------|------|----------|
| BTN_PWR  | GPIO0  | 电源键 | 支持（EXT0） |
| BTN_UP   | GPIO35 | 上键   | 不需要 |
| BTN_HOME | GPIO34（*） | Home 键 | 不需要 |
| BTN_DOWN | GPIO39 | 下键   | 不需要 |

> (*) GPIO34/35/39 仅支持输入，无内部上拉，需接 10kΩ 外部上拉电阻到 3.3V。

## 2. 接线指南

### 2.1 ESP32 与 4.2 寸三色墨水屏连接

墨水屏典型接口为 8 针（Waveshare 4.2" B 系列）：

| 墨水屏引脚 | 功能 | 接 ESP32 |
|-----------|------|----------|
| VCC | 电源 3.3V | 3.3V |
| GND | 地 | GND |
| DIN | SPI MOSI（数据输入） | GPIO23 |
| CLK | SPI 时钟 | GPIO18 |
| CS | 片选（低有效） | GPIO5 |
| DC | 数据/命令选择 | GPIO17 |
| RST | 复位（低有效） | GPIO16 |
| BUSY | 忙信号（高=忙） | GPIO4 |

当前代码中的唯一接线配置源为 `src/platform/esp_board_config.h`。如果硬件接线调整，先修改该文件，再同步本接线表和操作手册。

> **时钟频率**：三色屏初始化阶段使用 2MHz，数据传输使用 4MHz，不可超过。

完整 ESP32 SPI 引脚汇总：

```
ESP32 GPIO23 → EPD SDA
ESP32 GPIO18 → EPD SCK
ESP32 GPIO5  → EPD CS
ESP32 GPIO17 → EPD DC
ESP32 GPIO16 → EPD RST
ESP32 GPIO4  → EPD BUSY
ESP32 3.3V   → EPD VCC
ESP32 GND    → EPD GND
```

### 2.2 ESP32 与 MicroSD 卡模块连接

SD 卡模块（SPI 模式，独立 SPI 总线或共用 SPI 总线均可）：

| SD 模块引脚 | 功能 | 接 ESP32 |
|------------|------|----------|
| VCC | 电源 3.3V | 3.3V |
| GND | 地 | GND |
| MISO | SPI 主入从出 | GPIO19 |
| MOSI | SPI 主出从入 | GPIO23（与 EPD 共用，*） |
| SCK | SPI 时钟 | GPIO18（与 EPD 共用，*） |
| CS | 片选（低有效） | GPIO15 |

> **SPI 总线共用**：EPD 和 SD 可共用同一 SPI 总线（GPIO23/18），通过各自的 CS 引脚区分。操作时同一时刻只有一个 CS 拉低。  
> **EPD 传输时钟频率（2–4MHz）低于 SD（25MHz）**，切换设备时需重新配置 SPI 频率。

完整 SD 卡接线：

```
ESP32 GPIO19 → SD MISO
ESP32 GPIO23 → SD MOSI  (与 EPD DIN 共用)
ESP32 GPIO18 → SD CLK   (与 EPD CLK 共用)
ESP32 GPIO15 → SD CS
ESP32 3.3V   → SD VCC
ESP32 GND    → SD GND
```

### 2.3 按键接线

所有按键一端接 GPIO，另一端接 GND。GPIO34/35/39 需要外部上拉：

```
BTN_PWR:
  GPIO0  → 按键一端
  GND    → 按键另一端
  (GPIO0 内部有上拉，无需外接)

BTN_UP:
  GPIO35 → 按键一端 → 也接 10kΩ 电阻到 3.3V (外部上拉)
  GND    → 按键另一端

BTN_HOME:
  GPIO34 → 按键一端 → 也接 10kΩ 电阻到 3.3V (外部上拉)
  GND    → 按键另一端

BTN_DOWN:
  GPIO39 → 按键一端 → 也接 10kΩ 电阻到 3.3V (外部上拉)
  GND    → 按键另一端
```

### 2.4 电池与电量检测接线

```
锂电池(+) → TP4056 BAT+
锂电池(-) → TP4056 BAT-
TP4056 OUT+ → ESP32 3.3V 稳压模块输入 / 直接接系统 5V
TP4056 USB  → USB-C 充电接口

电量检测分压电路：
BAT(+) → 100kΩ → GPIO36(ADC) → 100kΩ → GND
(2:1 分压，4.2V满电 → ADC读到约2.1V，3.0V低电 → 读到1.5V)
```

> **注意**：ESP32 ADC 参考电压 1.1V，使用 11dB 衰减（`ADC_ATTEN_DB_11`）可测量最高约 3.1V。分压比应确保满电时不超过 3.1V。

### 2.5 完整接线总览图（文字版）

```
                    ESP32 N16R8
                 ┌─────────────────┐
        GND ────│GND         GPIO0 │──── BTN_PWR (另端→GND)
       3.3V ────│3.3V       GPIO34 │──── BTN_HOME + 10kΩ→3.3V
                │           GPIO35 │──── BTN_UP   + 10kΩ→3.3V
   EPD DIN ────│GPIO23      GPIO39 │──── BTN_DOWN + 10kΩ→3.3V
   EPD SCK ────│GPIO18      GPIO36 │──── 分压电路(电量检测)
    EPD CS ────│GPIO5             │
    EPD DC ────│GPIO17            │
   EPD RST ────│GPIO16            │
  EPD BUSY ────│GPIO4             │
                │                  │
    SD MISO ────│GPIO19            │
    SD MOSI ────│GPIO23 (共用)     │
     SD CLK ────│GPIO18 (共用)     │
      SD CS ────│GPIO15            │
                └─────────────────┘
```

## 3. 开发环境配置

### 3.1 工具链

| 工具 | 版本 | 说明 |
|------|------|------|
| PlatformIO | 最新版 | VS Code 插件 |
| 开发框架 | ESP-IDF（通过 PlatformIO 选择） | 使用 `framework = espidf` |
| ESP-IDF 版本 | 5.1.x 或 5.2.x | PlatformIO 托管版本 |

### 3.2 platformio.ini

```ini
[env:esp32-n16r8]
platform = espressif32
board = esp32dev
framework = espidf

; Flash & PSRAM
board_build.flash_size = 16MB
board_build.partitions = partitions_16mb.csv
board_upload.flash_size = 16MB

; PSRAM (ESP32 N16R8 = SPIRAM)
build_flags =
    -DBOARD_HAS_PSRAM
    -mfix-esp32-psram-cache-issue
    -DCONFIG_SPIRAM_SUPPORT=1
    -DCONFIG_SPIRAM_USE_MALLOC=1

; Monitor
monitor_speed = 115200
monitor_filters = esp32_exception_decoder
```

当前真机开发阶段已新增 PlatformIO + ESP-IDF 工程骨架：`app_main` 会初始化共享 UI 状态、400×300 framebuffer 和内置点阵字体，并渲染首帧到 ESP 显示适配层。`platform/epd_frame` 会把 framebuffer 打包为黑/红两个 15,000 字节的 1bpp 平面；`esp_display` 当前会初始化 GPIO/SPI、提供 reset/busy/command/data 原语，并在串口日志中输出打包字节数、黑/红像素统计和校验值。`esp_input` 会轮询 POWER/UP/HOME/DOWN 四个低电平触发按键，并把事件送入共享 `app_handle_button` 状态机，触发重绘和刷新。具体墨水屏 SPI 控制器、复位/忙线时序和刷屏波形待硬件信息确认后接入。

### 3.3 分区表（partitions_16mb.csv）


## 4. 系统架构

### 4.1 固件分层

```
┌──────────────────────────────────────────────────────┐
│                应用层（功能模块）                      │
│  阅读 · 天气 · 日历 · 游戏 · 英语 · 设置 · 关于       │
├──────────────────────────────────────────────────────┤
│              UI 页面状态机 (task_ui, Core 1)           │
│         主界面 ↔ 各功能页 (宫格选择 + 返回)            │
├───────────────┬──────────────┬───────────────────────┤
│  排版渲染引擎 │  按键状态机  │  事件总线              │
│  点阵字体     │  GPIO ISR +  │  FreeRTOS Queue        │
│  + 帧缓冲    │  esp_timer   │  + EventGroup          │
├───────────────┴──────────────┴───────────────────────┤
│             硬件抽象层 (HAL)                           │
│  EPD三色驱动(SPI) · SD(FATFS) · SPIFFS · NVS         │
│  ADC电量 · WiFi · SNTP · HTTP客户端                   │
└──────────────────────────────────────────────────────┘
```

### 4.2 FreeRTOS 任务规划

| 任务 | 核心 | 优先级 | 栈 | 说明 |
|------|------|--------|-----|------|
| `task_ui` | Core 1 | 5 | 8KB | 唯一操作 EPD 的任务 |
| `task_btn` | Core 0 | 10 | 2KB | 按键去抖与事件 |
| `task_net` | Core 0 | 4 | 6KB | WiFi · 天气轮询 · SNTP |
| `task_power` | Core 0 | 8 | 2KB | ADC 采样 · 休眠计时 |

### 4.3 帧缓冲设计（三色屏）

三色屏需要维护两个独立的 1bit 位图层：

```c
// 分配于 PSRAM
uint8_t fb_black[EPD_WIDTH * EPD_HEIGHT / 8];  // 15000 bytes，黑色层
uint8_t fb_red  [EPD_WIDTH * EPD_HEIGHT / 8];  // 15000 bytes，红色层

// 像素设置规则：
// 白色：fb_black[i] bit=0, fb_red[i] bit=0
// 黑色：fb_black[i] bit=1, fb_red[i] bit=0
// 红色：fb_black[i] bit=0, fb_red[i] bit=1
// （黑+红同时置1行为未定义，避免使用）
```

## 5. 主界面规格

项目总布局见 本目录下图片 eink_ui_mockups.png

### 5.1 布局

```
┌──────────────────────────────────────────────────┐
│  状态栏（20px 高，黑底白字）                       │
│  14:35  晴 26°C 北京       [信号图标] [电池78%] │
│  ←时间+天气摘要（左）        WiFi+电量（右）→      │
├──────────────────────────────────────────────────┤
│                                                   │
│   [阅读]   [天气]   [日历]   [游戏]               │
│                                                   │
│   [英语]   [设置]   [关于]                        │
│                                                   │
│   更多应用可扩展到下一屏                           │
│                                                   │
└──────────────────────────────────────────────────┘
```

### 5.2 状态栏（高 20–24px，全宽）

| 元素 | 位置 | 内容 |
|------|------|------|
| 时间 | 左起 8px | `HH:MM`，紧凑数字间距，避免下半部分裁切 |
| 天气摘要 | 时间后 | 天气状态 + 温度 + 城市，如 `晴 26°C 北京`（有天气数据时显示） |
| 信号图标 | 右侧 | 使用手机信号条样式，不显示粗糙文字 `WiFi` |
| 电量 | 最右侧 | 电池图标 + 百分比，必须完整显示；低于 20% 改用**红色**显示 |

### 5.3 功能宫格

- 布局：4 列 × N 行，当前 7 个功能按 4+3 排列；超出 8 个应用时扩展到下一屏。
- 每格尺寸：约 88 × 86px，避免卡片线框占用空间。
- 选中态：使用红色短线/侧边标记或反白强调，不使用厚重外框包住每个图标。
- 图标：清晰可辨的像素图标，参考常见移动端应用图标语义；图标本体不带外部线框。
- 标签：功能名称，14px 左右，居中显示在图标下方，中文不能重叠或被裁切。

### 5.4 刷新时机

主界面不自动定时刷新（E-Ink 省电），以下事件触发主界面重绘：
- 从任意功能页返回
- 首次开机显示
- WiFi 连接状态变化（下次用户操作时同步更新）

## 6. 功能模块规格

### 6.1 阅读模块

#### 书架列表页

```
┌──────────────────────────────────────────────────┐
│  状态栏                                           │
├──────────────────────────────────────────────────┤
│  书架 (3 本)                              [返回] │
├──────────────────────────────────────────────────┤
│ ▶ 三体               刘慈欣  1.2MB  ████░░ 45%  │  ← 选中（红色行）
│   百年孤独           马尔克斯 0.8MB  ██░░░░ 12%  │
│   活着               余华     0.4MB  ░░░░░░  0%  │
│   ...（翻页显示更多）                             │
└──────────────────────────────────────────────────┘
```

- 每页最多显示 5–6 本书（取决于字号设置）
- 进度条：宽 60px，黑色填充，未读部分为空白
- 选中行：整行背景浅红色
- 书架进度与阅读页当前页联动；从书架打开不同书籍时恢复对应书籍的当前页。
- 最近打开的书籍在书架中显示红色 `最近` 标记，方便快速识别上次阅读入口。
- 已添加书签的书籍在书架中显示红色书签标记。

#### 阅读页

```
┌──────────────────────────────────────────────────┐
│  三体 | 第三章·黑暗森林                  23/312 │  ← 状态栏（书名截断）
├──────────────────────────────────────────────────┤
│                                                   │
│  正文内容区                                       │
│  支持 TXT 格式（UTF-8/GBK 自动检测）              │
│  按排版设置渲染（字体/字号/行距/字间距/页边距）   │
│                                                   │
│  页面底部保持留白，不显示操作提示文字或底部进度条 │
│                                                   │
└──────────────────────────────────────────────────┘
```

- 书名超过 10 字时截断为 `三体...`，显示 `第N章` 章节信息
- 进度仅在顶部标题栏显示为 `当前页/总页数`；阅读页底部不显示进度条。
- 翻页操作：每次翻页触发一次全刷（约 15–20s），用户需等待
- 翻页等待时显示"刷新中..."提示字样（写入帧缓冲的第一步完成即可看到）
- 阅读菜单：阅读页短按 HOME 打开菜单；UP/DOWN 切换 `继续阅读 / 查看目录 / 添加书签 / 退出到书架`；HOME 执行当前项；POWER 关闭菜单但停留阅读页。
- 目录弹层：从阅读菜单选择 `查看目录` 打开章节目录；UP/DOWN 选择章节，HOME 跳转章节起始页，POWER 返回阅读菜单。
- 书签反馈：添加书签后菜单项显示为 `已加书签`。
- 书签位置按书籍记录，当前仿真阶段保存在运行时状态中，后续硬件阶段再持久化到 SD/NVS。
- 阅读页裸按 POWER 不直接跳回首页，避免误触中断阅读。

#### 支持格式

| 格式 | 支持 | 说明 |
|------|------|------|
| TXT | 完整 | UTF-8 / GBK 自动检测 |
| EPUB | 基础 | 提取纯文本，不支持图片/复杂排版 |
| PDF | 不支持 | ESP32 资源不足 |

#### 阅读分页算法

1. 按段落切分文本
2. 逐字符测量点阵字体宽度，确定换行点（中文字符边界、标点、空格）
3. 禁止行首标点（，。？！；：）
4. 段落不跨页，累积行高超过正文区高度时整段移至下一页
5. 生成页索引数组 `PageIndex[]`，持久化到 SD 卡对应 `.idx` 文件
6. 翻页时 O(1) 定位，无需重新排版

## 7. 按键交互设计

### 7.1 按键事件定义

| 事件类型 | 触发条件 |
|----------|----------|
| 短按 | 按下并在 500ms 内释放 |
| 长按 | 按下超过 800ms，每 500ms 重复触发 |
| 超长按 | 仅电源键，按下超过 2000ms |
| 组合按 | 两键同时按下（误触窗口 100ms） |

### 7.2 电源键（BTN_PWR）全场景映射

| 场景 | 操作 | 行为 |

## 8. 三色显示规范

### 8.1 颜色语义规则

三色墨水屏只有黑/白/红三色，红色应作为**强调/重要信息**的专用色，不可滥用：

| 颜色 | 使用场景 |
|------|----------|
| 黑色 | 主要文字、图标、边框、进度条、普通内容 |
| 白色 | 背景、未填充区域 |
| 红色 | 当前选中项、今日日期、低温/高 AQI 数值、低电量警告、食物/目标（游戏）、音标、重要提示 |

### 8.2 帧缓冲操作 API 规范

```c
// 像素级操作
void epd_draw_pixel(uint16_t x, uint16_t y, epd_color_t color);
// color: EPD_BLACK / EPD_WHITE / EPD_RED

// 矩形
void epd_fill_rect(uint16_t x, uint16_t y, uint16_t w, uint16_t h, epd_color_t color);
void epd_draw_rect(uint16_t x, uint16_t y, uint16_t w, uint16_t h, epd_color_t color);

// 文字（点阵字体）
void epd_draw_string(uint16_t x, uint16_t y, const char *str, uint8_t font_size, epd_color_t color);
void epd_draw_char(uint16_t x, uint16_t y, uint32_t unicode, uint8_t font_size, epd_color_t color);

// 图片（1bit BMP）
void epd_draw_bitmap(uint16_t x, uint16_t y, const uint8_t *bitmap, uint16_t w, uint16_t h, epd_color_t fg);

// 提交到屏幕（全刷，耗时 15-20s）
void epd_display();

// 清除帧缓冲为白色（不刷新）
void epd_clear_buffer();
```

### 8.3 刷新策略与用户体验

由于三色屏全刷耗时约 15–20 秒，需特殊处理：

1. **即时反馈**：用户操作后，先在帧缓冲写入"加载中..."提示字样，再开始正式渲染，最后一次性刷新
2. **阅读翻页**：提示"正在翻页..."，耐心等待约 15 秒
3. **主界面切换**：由于刷新慢，宫格光标移动时只更新帧缓冲，不立即刷新；按 HOME 进入功能时才触发刷新（加载画面）
4. **游戏操作**：每步操作后等待一次完整刷新，体验类似棋盘游戏（非实时）
5. **设置修改**：修改设置后，返回主界面时统一刷新一次

## 10. 电源管理

### 10.1 运行模式

| 模式 | 条件 | 预计电流 |
|------|------|----------|
| 活跃（WiFi 开） | 网络请求期间 | ~120–160mA |
| 活跃（WiFi 关） | 阅读/游戏/日历 | ~40–60mA |
| EPD 刷新中 | 刷新约 15–20s | +20mA（屏幕刷新功耗） |
| 深度睡眠 | BTN_PWR EXT0 唤醒 | < 0.1mA |

### 10.2 自动休眠逻辑

```
无操作计时器（task_power 中 esp_timer）
  ↓ 超过配置时间（默认 10 分钟）
保存当前状态到 NVS
  ↓
EPD 刷新一次"设备已休眠"提示画面
  ↓
关闭 SD 卡电源（GPIO 控制，可选）
  ↓
esp_deep_sleep_enable_gpio_wakeup(BTN_PWR_GPIO, ESP_GPIO_WAKEUP_GPIO_LOW)
esp_deep_sleep_start()
```

### 10.3 电量检测与显示

- ADC 采样：每 60 秒采样一次（16 次平均），task_power 执行
- SOC 映射（锂电池放电曲线）：

| 电压 | 电量 |
|------|------|
| ≥ 4.1V | 100% |
| 3.9V | 75% |
| 3.7V | 50% |
| 3.5V | 25% |
| 3.3V | 10% |
| ≤ 3.0V | 强制休眠 |

- 低于 20%：状态栏电量数字改为**红色**
- 低于 5%：全屏显示"电量不足，即将关机"→ 深度睡眠

## 11. 非功能性需求

| 指标 | 目标 |
|------|------|
| 全刷耗时 | 15–20s（三色屏固有限制） |
| 冷启动到主界面 | ≤ 3s（WiFi 连接异步进行） |
| 续航（深度睡眠） | ≥ 4 周（2000mAh，深睡 < 0.1mA） |
| 续航（阅读模式，WiFi 关） | ≥ 8–10 小时（2000mAh，每 5 分钟翻一页） |
| 书架容量 | SD 卡容量上限，书目索引支持 ≥ 500 本 |
| 词库大小 | 单 JSON 文件 ≤ 5MB（5000 词以内） |
| 崩溃恢复 | Task WDT 超时自动重启，重启后恢复阅读位置 |
| 堆内存 | 帧缓冲（30KB）分配于 PSRAM，主 SRAM 留 ≥ 32KB |
